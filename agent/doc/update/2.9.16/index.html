Изменения в версии 2.9.16

Оплата по QR-коду

В заявку добавлена поддержка тега [QRCode] для вывода QR-кода, по которому можно оплатить за тур, просканировав его в банковском приложении.

В QR-код попадают следующие реквизиты:

  • Название компании
  • ИНН
  • КПП
  • Номер расчетного счета
  • Название банка
  • БИК
  • Корреспондентский счет
  • Сумма к оплате
  • Назначение платежа

Размер QR-кода можно настраивать с помощью числового параметра в теге. Значением параметра могут быть 1 или 2. В случае единицы код будет сгенерирован минимального размера, в случае двойки – двойного размера. Тег [QRCode] без указания параметра эквивалентен [QRCode(2)].

QR-код в Счете

Добавлена возможность в Счете выводить QR-код для заполнения реквизитов платежа в банковских приложениях. В шаблоне счета для этого нужно использовать тег [QRCode] .

На форме счета добавлено поле "Назначение платежа", которое заполняется по образцу, задаваемому на вкладке "Счета" в настройках шаблонов. В шаблоне строки "Назначения платежа" можно использовать следующие теги:

  • НомерЗаявки
  • ДатаЗаявки
  • НачалоТура
  • КонецТура
  • Страна
  • ПервыйТурист
  • КоличествоТуристов
  • НомерСчета
  • ДатаСчета

По умолчанию поле "Назначение платежа" заполняется по следующему шаблону:

Оплата за тур по Счету № [НомерСчета] от [ДатаСчета]
